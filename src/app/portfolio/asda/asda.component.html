Project Description

Walmart is a retailing website that sells almost all the products both online and in stores. And in the current project the aim was to add new business requirements, update & enhance the existing functionalities like simplify adding to cart, increase or decrease the quantity, quick look at other relavent products, new click and collect feature, showing dynamic ads like new products and related products to consider, etc. Without effecting other modules and optimise the already existing website there by gradually revamping the entire child website but retaining the branding of the parent with enhanced user experience with strict web accessibility and as per W3C standards.

Learning & Takeaways

Get Early Clarifications Visual Design team being a thrid party there is a disconnect for the designs that we get from them for UI development due ot which a lot of re-work used to happen. When we found this we started implementing a strategy i.e. there will be a 2 weeks gap between the design approval and developemnt time which saved us re-work.

Prioritise TasksLike a ripple effect the time we spent on UI development affected the back-end development because of the dependency so we followed the same 2 week gap strategy so that there will not be any dependency from the FE team to BE team.

Manage Feedback Effective implementation of managing the feedback has saved projects overall timeline and we could deliver the project ontime without effecting the business.
